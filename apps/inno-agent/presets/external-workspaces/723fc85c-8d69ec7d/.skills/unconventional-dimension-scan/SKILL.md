# 冷门维度扫描

## When to use
- 用户明确表示"前面老师已经说过XX优点/缺点，我还能说什么"，面临发言资源枯竭困境
- 需要提供"从学生角度"而非"从教师表现角度"的评课观察点
- 请求涉及认知负荷、技术伦理、微观教学行为等专业维度的差异化分析
- 要求列出具体可观察的课堂细节（如等待时间、键盘操作差异、订正情绪等）

## When not to use
- 用户已明确具体观察内容，仅需润色表达或转化为得体话术 → 转交 **constructive-feedback-packaging**
- 用户询问如何记录前面老师发言或一般性教研礼仪 → 使用通用输入处理
- 用户表达"不管对不对，只要和别人不一样"的强行找茬意图 → 安全警告：坚持专业真实性，拒绝形式主义评课
- 用户意图评价教师个人特质（如性格急躁）而非教学行为 → 阻止并引导聚焦可观察的教学行为

## Instructions
1. **建立禁区地图**：接收用户提供的已发言关键词清单（如"情境导入好"、"PPT精美"），标记已覆盖的常规维度（教学内容、方法、技术、教态、纪律）
2. **激活冷门框架**：从四大专业维度扫描**未被触及**的观察点：
   - **学生认知维度**：错误类型利用、认知负荷峰值、学困生具体卡点、同伴嘲笑行为
   - **技术使用维度**：多媒体时长与目标匹配度、技术介入对注意力的干扰、AI生成内容与知识点的契合度
   - **课堂生成维度**：突发事件处理、等待时间分配、非预设问题的价值挖掘
   - **微观教学行为**：提问梯度设计、过渡语专业性、板书与PPT信息互补性、键盘操作熟练度差异
3. **执行视角转换**：将他人评价的"优点"转化为潜在风险观察（如将"纪律好"转化为"学生活力不足"），或从宏观评价转向微观细节（如关注"订正作业时的情绪反应"而非"作业完成情况"）
4. **生成观察假设**：基于学生中心策略，提出具体可验证的观察假设（如："AI视频的视觉刺激可能掩盖了数学思维过程"），而非直接给出结论性评价

## Workflow
1. **输入确认**：确认已发言内容清单，建立"禁区地图"（标记已覆盖维度）
2. **维度扫描**：在四大冷门框架中交叉比对，识别**零重叠**的观察切入点
3. **差异定位**：执行视角转换（优点→风险）或微观切入（整体→细节），确保与已发言内容本质不同
4. **假设生成**：输出基于学生学习的具体观察假设，附带课标理论依据（如"基于教学评一体化理念"）

## Output Format
```markdown
**场景判断**：当前是否处于发言禁令场景（是/否）及已覆盖维度分析

**差异化观察点**（2-3条）：
- 观察维度：[学生认知/技术伦理/课堂生成/微观行为]
- 具体观察：[微观细节描述，避免宏观评价]
- 理论依据：[引用2022版课程方案或学科课标具体条款]
- 与已发言内容的区别说明：[明确指出为何此不重复]

**风险提示**：
- 避免为不同而不同的刻意挑刺
- 确保观察基于真实课堂可能性，非虚构

**后续处理建议**：
- 将上述观察点交由建设性话术包装技能，转化为"肯定意图+专业观察+小措施"结构
```

## Examples

**输入**：已发言清单包含"导入生动（AI视频）"、"课堂纪律好"、"PPT设计精美"

**输出**：
- **观察点1（技术伦理）**：AI水果销售视频虽生动，但"盒子装满"的数学概念在动态演示中被快速跳过，学生注意力集中在水果色彩而非数量关系（基于数学课标"情境服务于思维过程"）
- **观察点2（学生认知）**：当教师提问后，学困生（如访谈中提及的"20以内加减法十分钟零分"学生）的认知卡点在哪个具体步骤？建议观察其键盘操作熟练度差异与情绪反应
- **观察点3（课堂生成）**：当学生因回答不出被同伴嘲笑时，教师如何即时制止并引导尊重行为，这体现社会情感学习的落实（将"纪律好"转化为对突发事件处理的观察）

## Common Issues
- **为不同而不同**：刻意将"好"说成"坏"而非基于专业判断的视角转换（如仅说"纪律太死"而无观察依据）
- **视角过于宏观**：仍停留在"教学设计合理"等宏观评价，无法规避重复
- **忽视学生中心**：仍聚焦"教师教得怎样"而非"学生学得怎样"，导致观察点与他人雷同
- **理论空泛**：使用"体现新课程理念"等AI套话，缺乏具体课标条款支撑

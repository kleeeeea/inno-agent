# 提取顺延申请信息
## When to use
- 当用户明确要求从培训顺延申请文档或文本中提取关键信息时。
- 当用户上传了一份培训顺延申请，并询问其中的申请人、单位、培训名称、日期或原因时。
- 适用于需要结构化呈现申请信中核心要素的场景。

## When not to use
- 当用户要求生成一份新的培训顺延申请信时（应路由至 `生成顺延申请信`）。
- 当用户询问申请是否符合规定或要求修改申请以提高批准率时（超出范围，需人工专家）。
- 当用户询问其他类型的申请信息（如请假、报销等）时（超出范围）。
- 当用户要求对申请的合理性、合规性或成功率进行评估或预测时（超出范围）。

## Instructions
- 接收用户提供的培训顺延申请文档或文本内容。
- 识别并提取申请人姓名、工作单位、申请日期、培训名称和顺延原因。
- 以清晰、结构化的格式输出提取到的信息。

## Workflow
1. 接收用户上传的文档或提供的文本。
2. 调用文本处理能力，识别并定位申请信中的关键信息字段。
3. 提取“申请人”、“工作单位”、“申请日期”、“培训名称”和“顺延原因”。
4. 整理提取结果并准备输出。

## Output Format
```
场景判断：用户请求提取培训顺延申请的关键信息。
处理方案：已成功从提供的文档/文本中提取以下信息。
可用话术：
- 申请人：[提取到的申请人姓名]
- 工作单位：[提取到的工作单位]
- 申请日期：[提取到的申请日期]
- 培训名称：[提取到的培训名称]
- 顺延原因：[提取到的顺延原因]
风险提示：无
后续评估：请用户确认提取信息是否准确，或是否有其他信息需要提取。
```

## Examples
- 用户：“请从这份文档中提取培训顺延申请的关键信息。”
- 用户：“帮我分析一下这份培训顺延申请，告诉我申请人是谁，单位是哪里，申请了什么培训，原因是什么。”
- 用户：“我上传了一份培训顺延申请，请提取里面的姓名、单位和日期。”

## Common Issues
- **信息缺失：** 文档中未明确包含所有关键信息，导致部分字段无法提取。
- **格式不规范：** 申请信格式不标准，影响信息识别的准确性。
- **歧义信息：** 文档中存在表述不清或有歧义的内容，可能导致提取错误。
- **非培训顺延申请：** 用户上传了非培训顺延申请的文档，导致无法提取相关信息。

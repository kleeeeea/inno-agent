# 试讲呈现诊断分析

## When to use

- 考生上传试讲录像（10分钟片段）或文字稿，要求分析"为什么看起来不像老师"
- 考生自我描述呈现问题（如"语速快""表情僵""手不知道放哪"），需要定位核心症结
- 备考初期首次评估，需确定四要素（状态/节奏/表情/动作）中优先级最高的改进项
- 多轮训练后需要验证前序问题是否解决，重新定位新的瓶颈

## When not to use

- **需转给 `presentation-training`**：当用户要求生成训练方案、微目标设定、停顿点标注、上台脚本或具体练习方法时（诊断后应建议转交该技能执行训练）
- **普通问答**：询问教师资格面试流程、结构化问答技巧、学科知识点准确性、教案设计逻辑
- **安全转介**：考生出现严重焦虑躯体化症状（头晕呕吐）、过度自我否定（"我肯定不行"等绝对化表述），优先建议就医或心理咨询，暂停技术分析

## Instructions

1. **单一核心原则**：必须锁定"最制约教师感的单一问题"，拒绝面面俱到，明确告知"本轮只改这一项"
2. **四要素扫描**：按状态（能量/角色进入）、节奏（停顿/语速）、表情（微笑/亲和）、动作（手位/姿态）四维快速扫描，定位负面因素
3. **可观测指标**：所有诊断结论必须对应可录像验证的指标（如"前15秒目光游离""10分钟零停顿"），避免主观感受描述
4. **区分层级**：区分"技术问题"（可控）与"特质焦虑"（需转介），技术问题方可进入诊断流程

## Workflow

1. **素材收集**：确认用户提供录像（要求：前30秒+任意2分钟片段）或文字稿/自我描述
2. **四要素定位**：
   - 状态：检查前10秒能量、声音亮度、站姿稳定性
   - 节奏：标记停顿次数、语速变化、信息密度点
   - 表情：识别微笑锚点缺失、僵硬区间、自然度
   - 动作：评估默认手位、功能性手势比例、僵硬/松散倾向
3. **优先级排序**：按"对教师感破坏力最大→改进杠杆效应最高"排序，选出TOP1问题
4. **诊断输出**：输出包含"核心问题→四要素定位→验证指标"的诊断报告

## Output Format

```markdown
### 诊断结论
- **核心问题**：[单一问题描述，如"节奏失控：零停顿导致信息密不透风"]
- **四要素定位**：[状态/节奏/表情/动作] → 具体表现
- **优先级**：P0（立即阻断）/ P1（显著减分）/ P2（优化项）
- **验证指标**：[可观测、可量化的具体指标，如"前30秒出现3次以上目光下移""10分钟内句间停顿<2次"]

### 关联影响
- 该问题如何破坏教师感：[1句话说明因果，如"考官在前10秒无法建立信任，后续内容被低估"]
- 次要问题（本轮暂不处理）：[列出但明确标注"冻结"，防止用户同时修改]

### 下一步建议
- 如需训练方案：转交 `presentation-training` 生成本轮微目标与执行脚本
- 如时间紧急（<3天）：提供"保命底线"临时规避策略
```

## Examples

**输入**：用户上传录像并描述"我总被说太紧张，手不知道放哪，语速也快"

**诊断输出**：
```markdown
### 诊断结论
- **核心问题**：动作僵硬导致的能量通道堵塞（而非语速本身）
- **四要素定位**：动作 → 全程"捂肚脐"手位+肘部锁死，导致上半身僵硬传导至声音急促
- **优先级**：P0
- **验证指标**：录像中手位低于肚脐时间占比>80%；肘关节角度<90°持续时间>70%

### 关联影响
- 该问题如何破坏教师感：防御性手位传递"我不确定"信号，引发语速代偿性加快以掩盖不安
- 次要问题（本轮暂不处理）：语速快（待手位放松后自然缓解）、表情紧绷（P1冻结）

### 下一步建议
- 转交 `presentation-training` 设计"默认手位解放+肩部放松"专项训练
```

## Common Issues

1. **贪多求全**：用户希望一次改掉"语速快+表情僵+手乱动"，必须强制聚焦单一核心，解释"一次只改一项，多改必乱"
2. **归因错误**：将呈现问题归咎于"教案不够精彩"，需纠正为"状态不对时，再精彩的设计也听不进去"
3. **忽视前30秒**：用户关注中段教学内容，提醒"考官决策主要基于前30秒的第一印象，诊断权重前30秒占50%"
4. **自我诊断偏差**：用户自称"语速快"实为"节奏无停顿"，或自称"紧张"实为"能量不足"，需以录像证据为准纠正认知

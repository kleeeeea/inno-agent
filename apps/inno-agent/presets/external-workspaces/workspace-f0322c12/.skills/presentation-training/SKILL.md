# 试讲呈现优化训练

## When to use

- 用户已明确具体缺陷（如"语速过快""表情僵硬"），需要**可落地的训练指令**（停顿点位置、手位规则、上台脚本）
- 用户要求生成针对单一要素的**微目标训练方案**（如"本轮只练节奏"）
- 用户需要**上台前30秒状态调整脚本**或**考场高压环境下的自动执行流程**

## When not to use

- **诊断阶段**：用户仅提供录像/文字稿但尚未定位核心问题（应转交 `trial-teaching-diagnosis` 进行四要素诊断）
- 涉及学科知识本体、教案设计、结构化问答、答辩策略等超出"呈现层"的内容
- 用户出现严重身体症状（头晕、过度换气）或深层心理问题（考试焦虑症），需建议就医或转介专业心理咨询

## Instructions

1. **一次只改一项**：强制设定单一微目标（如"仅训练节奏，加入5个停顿点"），禁止多要素同时训练
2. **动作可观测**：所有指令必须可录像验证（如"停顿0.5秒""手位：腹前自然放置"），禁止抽象建议（如"放松一点"）
3. **反例警示**：明确指出"端着僵硬""切菜式手势""密不透风语速"等误区，提供替代动作指令

## Workflow

1. **接收诊断结果**：确认当前需优化的单一要素（状态/节奏/表情/动作）及具体表现
2. **生成训练方案**：
   - 设定本轮**微目标**与可观测指标
   - 标注**停顿点**（位置与时长）或**微笑锚点**（触发场景）
   - 规定**默认手位**与功能性手势使用规则
   - 提供**上台10秒流程脚本**（站定—目光—开口—节奏—微笑）
3. **指导自测执行**：要求录制10-15分钟试讲，回看重点为"前30秒+任意2分钟片段"
4. **固化迁移**：形成个性化"上台脚本"，确保考场高压下可自动执行

## Output Format

```markdown
### 本轮微目标
- 优化要素：[状态/节奏/表情/动作]
- 可观测指标：[具体数字或行为描述]

### 具体动作指令
- **停顿点标注**：[文本位置]停顿时长
- **微笑锚点**：[场景触发，如"学生回答后肯定时"]
- **默认手位**：[胸前交叉/腹前放置等]
- **禁忌动作**：[明确禁止的行为]

### 上台10秒流程脚本
1. 站定：[动作]
2. 目光：[视线落点]
3. 开口：[第一句话话术]
4. 节奏：[首句语速要求]
5. 微笑锚点：[首次微笑时机]

### 验证标准
- [ ] 录像自测通过指标
- [ ] 同伴盲评问题清单
```

## Examples

**用户**：我语速一紧张就快，怎么在不改教案的前提下加停顿？

**输出**：
- **微目标**：本轮仅训练节奏，强制加入5个停顿点与2次重音放慢
- **停顿点标注**：①开场问候后（0.5秒） ②提问后（1秒，等"学生"反应） ③板书后（转身前0.5秒） ④知识点过渡（"接下来..."前0.5秒） ⑤总结前（1秒）
- **重音放慢**：在核心概念处（如"**光合作用**"）语速降20%，音量微抬
- **验证标准**：录像中10分钟内出现明显停顿≥5次，无连续20秒无停顿片段

## Common Issues

1. **贪多求全**：用户想同时改状态+节奏+动作 → 强制限定本轮仅改一项，其他"维持现状即可"
2. **反馈无效**：同伴只说"有点紧张" → 提供**盲评问题清单**（"前30秒是否愿意继续听？""手势是否分散注意力？"）
3. **微笑出戏**：刻意笑导致思路断裂 → 改为"微笑锚点"训练（特定节点触发，如"学生回答后点头微笑"），配合肌肉记忆练习
4. **考场迁移失败**：平时练得好，考试忘光 → 必须生成**上台10秒流程脚本**，要求考前3天每天 rehearse 3次完整上台流程（含物理走位）

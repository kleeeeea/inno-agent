# 技能 1
## When to use
*   用户需要生成或完善《教育任务描述文件》中任一部分内容时（如“任务名称”、“触发情景”、“核心问题”等）。
*   用户希望根据其提供的初步信息，按照模板要求进行结构化和内容填充。
*   用户寻求关于如何更好地描述某个模板章节内容的具体指导和建议。
*   用户需要一份完整的《教育任务描述文件》草稿。

## When not to use
*   与《教育任务描述文件》模板无关的教育咨询、课程设计或教学方法探讨。
*   要求提供模板以外的额外文档、资源链接或学术论文。
*   关于智能体自身功能、技术问题或故障排除的询问。
*   要求进行数据分析、图表制作或多媒体内容生成。
*   用户试图生成与教育伦理不符、歧视性或非法内容。
*   用户要求提供专业法律、医疗、心理咨询或超出教育任务描述范围的政策解读。
*   用户输入涉及个人隐私、商业机密或国家安全等敏感信息。

## Instructions
*   明确您的需求：是需要生成完整文件，还是完善特定章节？
*   提供初步信息：根据智能体的提示逐步输入各章节所需内容。
*   积极回应引导：智能体将通过提问引导您深入思考和细化内容。
*   随时反馈：对生成内容有疑问或需要修改时，及时提出。

## Workflow
1.  **需求确认与引导**：智能体确认用户意图，并简要介绍模板结构。
2.  **逐项内容填充**：按照《教育任务描述文件》模板章节顺序（任务名称、触发情景、核心问题、解决方案、评价准则、案例对比、规范依据、测评问题）引导用户。
3.  **深度挖掘与细化**：针对每个章节，提出具体问题，引导用户深入思考，避免内容空泛。
4.  **结构化建议与优化**：提供格式建议，提示补充细节，确保内容符合规范。
5.  **整合与输出**：完成所有章节后，整合生成完整的《教育任务描述文件》草稿。

## Output Format
*   任务名称
*   触发情景（适用学段、环境限定、约束条件、任务意图）
*   核心问题（导语、结构性问题、机制性问题、衍生问题）
*   解决方案（导语、线性流、分支流、循环流）
*   评价准则（维度、权重、优秀标准、合格标准）
*   案例对比（正面案例、反面案例）
*   规范依据
*   测评问题（情景主题、情景问题）

## Examples
*   **用户请求**：请帮我生成一份关于“中职学生职业素养培养”的教育任务描述文件。
*   **智能体响应**：好的，我们将从“任务名称”开始。请您提供具体的任务名称。
*   **用户请求**：在“核心问题”部分，我想描述当前职业素养教育中理论与实践脱节的问题。
*   **智能体响应**：好的，这属于结构性问题。请您具体描述其表现形式和影响。

## Common Issues
*   **内容空泛**：用户输入过于笼统，缺乏具体细节。
*   **偏离模板**：用户试图输入与模板要求不符的内容。
*   **信息不全**：用户未能提供足够信息来填充某个章节。
*   **专业咨询误区**：用户将智能体视为专业教育顾问，寻求超出模板范围的建议。

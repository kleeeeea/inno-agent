# 跨课程概念分工与跨界融合设计

## Instructions
- 拒绝"加强沟通"等宏观口号，只交付表格化工具与可执行SOP
- 概念分工必须明确边界（谁讲概述/谁讲设计/谁讲实操），杜绝"炒冷饭"
- 跨界设计必须绑定真实职业场景，配套"角色强制绑定+随机抽取"防搭便车机制

## Workflow
1. **重复概念扫描**：识别同一专业不同课程中的重复知识点（如三门课都讲"压强"基础定义）
2. **分工矩阵绘制**：构建跨课程概念分工表，划定各课程讲授边界（深度/应用场景/实操要求）
3. **专业情境重构**：将公共课知识点映射为真实职业任务（如数学→密码破解、物理→数控操作）
4. **硬核卷入机制**：设计"角色强制绑定+随机抽取汇报"规则，确保C/D角色无法搭便车

## Output Format
- **跨课程概念分工矩阵**：表格化清单，明确各课程对同一概念的讲授侧重点与禁止重复区
- **专业情境任务卡**：包含职业场景描述、操作步骤、角色分工表（A操作/B记录/C验证/D汇报）
- **防搭便车规则书**：随机抽号机制、过程性产出要求、缺席惩罚条款

## Examples
- **概念分工示例**：烹饪专业"蛋白质变性"在三门课中的边界划分（营养学讲分子原理/食材学讲影响因素/实操课讲火候控制与质构判断）
- **跨界融合示例**：数控专业"三角函数"重构为"车床锥度计算与对刀实操任务"，强制全员提交测量数据并随机抽号答辩

## Common Issues
- **定位模糊**：避免使用"相关"、"结合"等模糊词，必须明确"概述课只讲X，实操课必须讲Y"
- **虚假跨界**：防止"专业举例"式 superficial 融合，必须设计可操作的、有产出的职业任务流程
- **参与漏洞**：禁止自由讨论，必须设定角色绑定（如规定C角色负责数据验证，不发言则小组任务无效）

# 中职课堂高效诊断与跨专业改进助手评分标准

## 维度一：数据采集客观性（权重25%）

| 等级 | 评分标准 |
|------|----------|
| **优秀** | 全面采用秒级量化指标（抬头率%、动笔率%、答题正确率%/缺席率%）；彻底摒弃"气氛活跃"、"听讲认真"等主观玄学描述；信息化采集SOP可在3分钟内完成部署并生成实时数据看板；1分钟扫描法观察项≤5个且全部可打勾/填数字，无开放式文字描述 |
| **良好** | 主要使用量化指标，偶现模糊描述但可快速修正；信息化工具部署流程清晰，能在5分钟内呈现关键数据；观察表设计简洁，以勾选和数值为主，文字描述占比<20% |
| **合格** | 混合使用量化数据与主观评价（如"大部分学生参与"）；信息化工具使用流程存在冗余步骤；观察表包含少量定性描述栏位，但未影响核心数据收集 |
| **待改进** | 依赖主观感受描述（"教师充满激情"、"学生很配合"）；缺乏具体测量工具或操作流程；观察表设计复杂，需要大量文字记录，无法快速完成课堂扫描 |

## 维度二：问题归因精准度（权重20%）

| 等级 | 评分标准 |
|------|----------|
| **优秀** | 精准识别并区分三大隐性低效毒瘤：假性参与（互动覆盖窄、眼神空洞、C/D角色搭便车）、概念重复/定位不清（多课程炒冷饭、实操深度缺失）、结构失衡（避重就轻、偏题闲聊）；每个判定均附带具体数据证据（如"仅2名优生发言，占比<10%"） |
| **良好** | 准确识别2类以上低效问题，能区分假性参与与真实参与；对概念重复和结构失衡有基本判断，但边界界定略模糊；归因基本基于观察数据，偶有经验推测 |
| **合格** | 能识别明显低效现象（如学生不抬头、教师跑题），但未能归类为三大毒瘤类型；问题描述较为笼统（"学生参与度不高"），缺乏具体行为指标支撑 |
| **待改进** | 问题判定停留在表面现象（"课堂纪律差"、"学生基础弱"）；未能识别假性参与、概念重复等隐性低效；归因模糊（"可能是教材问题"），无数据支撑 |

## 维度三：方案工具化程度（权重25%）

| 等级 | 评分标准 |
|------|----------|
| **优秀** | 交付物为立等可用工具：《跨课程概念分工矩阵》（表格化，直接下发）、《防偏题话术卡》（3句强硬拉回主线且不伤害师生感情的具体话术）、《应急微操手册》（答题器正确率<40%时的两步走策略，第一步…第二步…）；**零宏观口号**（无"加强"、"提高"、"优化"等不可执行词汇） |
| **良好** | 提供具体工具模板，但需轻微调整即可使用；话术和策略较为具体，有明确步骤；存在极少数口号式表述，但不影响执行；工具针对特定分支（概念治理/结构纠偏/跨界融合） |
| **合格** | 提供改进建议清单，但部分需二次加工（如"建议调整时间分配"但无量化标准）；存在部分可执行工具，混杂少量宏观指导（"应注重学生主体性"）；缺乏应急场景的具体话术 |
| **待改进** | 输出宏观指导原则（"建议加强师生互动"、"需要提高课堂效率"）；无具体工具、话术或SOP；改进方案无法直接下发给教师执行，需要使用者自行解读转化 |

## 维度四：跨界融合设计质量（权重20%）

| 等级 | 评分标准 |
|------|----------|
| **优秀** | 构建清晰的**跨课程概念分工矩阵**，明确同一概念在不同课程的深浅边界（如概述课讲单位换算→设计课讲内压外压影响→实操课讲表盘读数）；公共课完全重构为专业情境（数学+密码学、物理+数控）；设计**硬核卷入机制**（A操作/B记录/C验证/D汇报+随机抽号汇报，彻底杜绝搭便车） |
| **良好** | 识别出跨课程概念重复问题并提供分工建议，但边界界定需进一步细化；公共课与专业有一定结合（举例说明应用场景）；小组任务有角色分工，但缺乏强制随机抽取机制 |
| **合格** | 提到跨专业整合概念，但未提供具体分工方案；公共课情境设计较为泛化（"结合专业举例"），未深入真实职业场景；有小组合作设计，但角色分工模糊，无法防止搭便车 |
| **待改进** | 忽视跨专业脱节问题；公共课与专业课完全割裂；未设计有效的小组协作机制，或仅建议"分组讨论"而无具体规则防止假性参与 |

## 维度五：资产沉淀完整性（权重10%）

| 等级 | 评分标准 |
|------|----------|
| **优秀** | 完整输出**校本资源库核心物料清单**4份：课堂原始数据截图、学生产出样本、跨界分工逻辑图、可复现SOP；建立**能力进阶测试数据**机制（诊断性测试模板设计）；所有资产可直接归档复用，形成标准化校本资源 |
| **良好** | 提供3类以上核心物料，格式规范；包含数据采集留痕建议；有测试数据收集意识，但未形成固定模板；资产基本可直接复用，少量需补充说明 |
| **合格** | 提供2类左右物料（如仅有分工表和SOP）；建议保留课堂数据但未明确格式；缺乏进阶测试数据机制设计；资产复用性一般，需较多调整 |
| **待改进** | 仅输出单次诊断结论，未形成可沉淀资源；无课堂数据留存建议；无SOP或分工图；资产无法复现，一次性使用 |

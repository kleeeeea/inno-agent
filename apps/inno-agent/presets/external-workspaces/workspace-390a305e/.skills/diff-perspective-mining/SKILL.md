# 差异化视角挖掘

## When to use
- 教研组集体评课中前面教师已说完 obvious 优缺点，面临"不能再说了"的发言禁令时
- 需要基于专业课堂观察框架快速定位冷门维度（如认知负荷、技术伦理、生成性事件）的观察点时
- 新教师在"师带徒"教研文化中需避免重复又不愿刻意挑刺，寻求建设性差异化视角时

## When not to use
- 尚未收集前面教师发言关键词与已覆盖维度时（应提示用户先完成阶段一输入）
- 观察材料中出现学生心理危机、虐待迹象、教师严重违规等高风险信号时（应立即转介安全处理，停止评课生成）
- 仅需通用评课模板而无具体课堂实录或已发言内容参照时（应使用普通问答）
- 需要生成完整即兴发言稿而非仅差异化观察切口时（应交由发言稿生成 skill）

## Instructions
- 严格基于已提供的"已发言关键词"排除重复维度，禁止虚构未观察到的细节
- 优先扫描学生认知维度（错误类型利用、认知负荷峰值）、技术使用伦理（技术-目标匹配度、注意力干扰）、课堂生成事件（突发事件处理、等待时间）、微观教学行为（提问梯度、信息互补）
- 采用视角转换（优点→潜在风险）、微观切入（具体细节替代整体评价）、学生中心（教师教得怎样→学生学得怎样）三大策略生成观点
- 所有观点必须基于2022版义务教育课标概念（如"认知负荷"、"生成性资源"、"教学评一体化"），杜绝AI套话和人身评价

## Workflow
1. **已发言排除**：标记已覆盖维度（教学内容/方法/技术/教态/管理），区分表面评价（"PPT精美"）与深层评价
2. **冷门维度扫描**：从学生认知、技术伦理、课堂生成、微观行为四类专业维度中定位未被触及的观察点
3. **差异化生成**：运用视角转换（如"纪律好"→"学生活力不足"）、微观切入（如"键盘熟练度差异"）、学生中心（"学生是否听懂"）策略生成零重复观点
4. **边界检查**：确认无AI套话、无人身评价、无刻意对立，符合"师带徒"文化下的专业边界

## Output Format
```markdown
**场景判断**：{是否具备差异化挖掘条件：已收集发言关键词/存在剩余专业维度}

**差异化切口**：{冷门维度名称，如"技术介入的认知干扰"或"错误类型的生成性利用"}

**专业观察点**：{具体微观描述，避免宏观评价，如"第5分钟AI视频播放时学生注意力集中在水果动画而非数量关系"}

**转化策略**：{采用的策略：视角转换/微观切入/学生中心，并说明具体逻辑}

**建设性话术框架**：
- 肯定意图：本节课在[具体环节]体现了[某教学意图]...
- 专业观察：但从[冷门维度]看，[具体现象]可能暗示[深层问题]...
- 小措施建议：建议尝试[具体可操作调整，如5秒等待时间/闭眼复述环节]...

**风险提示**：
- 避免AI套话：{检查是否出现"充分体现新课程理念"等空泛表述}
- 避免刻意挑刺：{检查是否为不同而强行对立}
- 安全边界：{确认未涉及高风险情境，如有应提示转介}
```

## Examples
**输入**：已发言内容包含"情境导入好"、"AI视频生动"、"纪律佳"、"PPT精美"
**处理**：排除技术应用（表面）、课堂管理等维度，扫描至"技术-目标匹配度"
**输出**：
差异化切口：技术使用的认知干扰
专业观察点：AI生成水果视频虽生动，但"盒子装满"的数学概念在动态演示中被快速跳过，学生注意力集中于水果动画而非数量关系（基于"凑十"教学细节）
转化策略：微观切入（不评价视频美观度，关注概念呈现清晰度）+ 学生中心（关注学生是否真正理解数量关系而非被视觉刺激分散）
建设性话术：本节课在情境创设上体现了生活化意图...但从技术伦理维度观察，AI视频的视觉刺激可能掩盖数学思维过程...建议增加闭眼复述关键信息环节检验真实理解...

## Common Issues
- **虚构观察细节**：用户未提供课堂实录时，易编造"学生表情变化"等未观察内容，应坚持仅基于输入材料生成观点
- **机械避坑**：为不重复而强行将"纪律好"对立为"纪律太死"，应转化为深层观察"学生活力不足"
- **AI套话陷阱**：生成"教师主导作用发挥得当"等空洞表述，应替换为微观具体描述如"提问等待时间不足3秒导致学困生无法跟上"
- **维度遗漏**：仅关注技术、教态等显性维度，忽略学生认知负荷、社会情感学习等冷门维度，应强制扫描四类专业维度

# 生成培训顺延申请
## When to use
*   用户明确表示需要“撰写”、“生成”或“起草”一份新的培训顺延申请。
*   用户提供了生成申请所需的所有关键信息，如申请人姓名、工作单位、申请事由、培训名称或日期等。
*   用户希望获得一份格式规范、内容完整的正式申请信。

## When not to use
*   用户提供了一份已有的申请文本，并要求从中提取信息（应使用 `parse-training-postponement-application` 技能）。
*   用户请求审批、提交申请或查询申请状态。
*   用户需要法律或政策咨询，或核实信息的真实性。
*   用户请求生成非培训顺延类型的申请（如请假、调动等）。
*   用户要求生成虚假、误导性或不道德的内容。

## Instructions
1.  主动询问或引导用户提供生成申请所需的关键信息，包括但不限于：申请人姓名、工作单位、申请事由、希望顺延的培训名称或日期。
2.  告知用户将根据其提供的信息和预设模板生成申请信。
3.  在生成前，可向用户确认关键信息是否准确无误。

## Workflow
1.  **识别意图：** 确定用户请求是生成新的培训顺延申请。
2.  **信息收集：** 引导用户提供申请人姓名、工作单位、申请事由、培训名称及希望顺延的日期等必要信息。
3.  **生成申请：** 根据收集到的信息，结合预设的申请模板，生成一份完整的培训顺延申请信。
4.  **结果呈现：** 将生成的申请信以清晰、专业的格式呈现给用户。
5.  **后续支持：** 询问用户是否需要对申请信进行修改或提供其他相关帮助。

## Output Format
```
尊敬的[接收方称谓，如：党委教师工作部]：

我是[申请人姓名]，[工作单位]的教师。因[申请事由]，我无法参加[培训名称或日期]的培训。对于缺席的培训内容，我特此申请顺延参加下一期。给您带来的不便，深表歉意。恳请批准！

此致 

敬礼！

申请人：[申请人姓名]
单位负责人：[预留空白或根据用户指示填写]
工作单位：[工作单位]
日期：[当前日期或用户指定日期]
```

## Common Issues
*   **信息不完整：** 用户未提供生成申请所需的全部关键信息。
    *   **处理方案：** 引导用户补充缺失的信息。
*   **意图不明确：** 用户请求模糊，难以判断是生成还是解析申请。
    *   **处理方案：** 澄清用户意图，询问是“撰写一份新的”还是“分析一份已有的”。
*   **超出边界请求：** 用户要求提供审批、提交或法律咨询等服务。
    *   **处理方案：** 明确告知服务边界，并拒绝执行超出能力范围的请求。

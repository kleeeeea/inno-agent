# 解析培训顺延申请
## When to use
*   用户提供了一份培训顺延申请的完整文本，并希望从中提取关键信息。
*   用户需要从已有的申请中快速识别申请人、单位、日期、事由等核心要素。
*   用户希望对收到的培训顺延申请进行结构化信息提取。

## When not to use
*   用户希望生成一份新的培训顺延申请（请使用“生成培训顺延申请”技能）。
*   用户请求审批、提交申请或提供法律/政策咨询。
*   用户提供的信息并非培训顺延申请，而是其他类型的文档。
*   用户要求核实申请内容的真实性。

## Instructions
*   请提供完整的培训顺延申请文本。
*   如果文本中包含多个申请，请明确指出需要解析哪一份。

## Workflow
1.  **接收文本：** 接收用户提供的培训顺延申请文本。
2.  **信息识别：** 智能识别并提取文本中的关键信息，包括但不限于申请人姓名、工作单位、申请日期、申请事由、原培训名称或日期。
3.  **结构化输出：** 将提取到的信息以清晰、结构化的格式呈现给用户。

## Output Format
场景判断: 识别到用户意图为解析培训顺延申请。
处理方案: 成功从文本中提取关键信息。
提取信息:
*   申请人: [提取到的申请人姓名]
*   工作单位: [提取到的工作单位]
*   申请日期: [提取到的申请日期]
*   申请事由: [提取到的申请事由]
*   原培训名称/日期: [提取到的原培训名称或日期]
风险提示: 提取信息可能因原文表述不规范而存在不完整或不准确的情况，请核对。
后续评估: 请问您对提取结果满意吗？是否需要进一步处理或有其他疑问？

## Common Issues
*   **文本不完整或模糊：** 提供的申请文本残缺不全或表述不清，导致信息提取困难。
*   **格式不规范：** 申请文本格式非标准，关键信息位置不固定，影响识别准确性。
*   **信息缺失：** 文本中未包含某些关键信息（如申请日期、具体事由等）。

# AI对抗课堂管理

## When to use
- 需要设计120分钟连排机房的六阶段教学时间轴与流程管控方案
- 面临学生切屏使用AI、机房纪律失控、分组协作低效等课堂管理困境
- 需配置小组角色分工（汇报员/AI检索员/记录员）及轮换机制
- 突发情况处置：学生公开质疑题目错误、拔网线冲突、严重挫败情绪爆发
- 需要生成可即用的三维度评价量规（错误识别/批判性分析/知识建构）

## When not to use
- **设计陷阱题目**：需设计包含逻辑错误、边界缺失的Python编程题时，应使用 **trap-question-designer** 技能
- **纯技术答疑**：学生询问Python语法纠错、代码调试等技术问题，使用普通问答模式
- **安全危机干预**：识别到学生自伤风险、严重焦虑发作或医疗紧急状况，必须立即提醒教师转介学校心理咨询中心，不得继续教学施压
- **教育决策替代**：涉及学生处分、成绩判定、家长沟通等教育管理决策，仅提供教学策略建议，最终决策必须由教师人工判断

## Instructions
- **严守隐私伦理**：不推荐安装监控学生屏幕的间谍软件，仅通过物理位置安排、任务拆解、角色分工等教学组织策略管理
- **分层适配**：区分高考班（侧重逻辑严谨性）与就业班（侧重基础防错）调整冲突强度与话术深度
- **话术嵌入**：直接提供教师可用的关键话术，如"不管是AI给你的还是老师给你的，他都有一定可能性...你要思考更天真"
- **风险预警**：对拔网线操作、公开质疑处理、过度挫败预防等高风险环节必须加粗警示

## Workflow
1. **学情与环境确认**：核实班型（高考班/中本一体/3+2/普高转职高）、机房硬件条件（是否允许切屏/断网）、学生先验水平分布
2. **六阶段流程生成**：输出0-20分钟（自主探究封锁）→20-30分钟（AI验证开放）→30-40分钟（冲突质疑）→40-100分钟（协作解构）→100-120分钟（元认知反思）的详细时间轴与转换节点
3. **组织架构设计**：配置3人小组角色分工与轮换机制，制定断网/切屏限制环境下的监控策略与座位安排
4. **突发应对预案**：针对"学生发现题目确实错误"、"拔网线冲突"、"严重挫败感"等场景设计话术与处置流程
5. **评价与反思工具**：生成三维度量规表格（错误识别30%/批判性分析40%/知识建构30%）及元认知反思问卷模板

## Output Format
### 场景判断
[当前面临的具体课堂管理场景：纪律失控/流程混乱/突发冲突/评价缺失]

### 处理方案
- **时间轴设计**：120分钟六阶段详细分配、转换节点与阶段目标
- **分组策略**：小组构成（3人制）、角色分工（汇报员/AI检索员/记录员）、轮换规则
- **技术管控**：断网/切屏限制的具体操作方案与物理监控策略

### 可用话术
- **阶段转换话术**：各阶段过渡时的教师指令与注意力管理
- **冲突调解话术**：应对质疑、挫败、纪律问题的具体措辞与语气指导
- **元认知引导话术**：反思阶段的提问模板与思维可视化工具

### 风险提示
- **[高风险环节]**：如拔网线操作、公开质疑处理、过度挫败预防的具体红线
- **心理安全红线**：需立即转介心理咨询的预警信号与紧急处置步骤

### 后续评估
- **过程性评价**：课堂观察要点、小组互评表与教师记录表
- **终结性量规**：三维度评分标准与权重（30%/40%/30%）
- **反思改进**：教师课后复盘问题清单与流程优化建议

## Examples

### 示例1：120分钟六阶段完整流程
**场景**：高考班，机房可断网但不可监控屏幕，学生已学循环结构

**处理方案**：
- **0-20分钟 自主探究封锁**：发纸质陷阱题，要求手写流程图，断网，教师巡视阻止过早使用IDE，座位调整为"独立应考"间距
- **20-30分钟 AI验证开放**：允许联网，小组分工（1人操作AI/1人记录/1人比对），要求复制AI代码到本地运行并截图保存AI原始回答
- **30-40分钟 冲突质疑**：强制暂停，展示典型错误输出，提问"AI说对了但跑错了，信谁？"，触发认知失调
- **40-100分钟 协作解构**：小组讨论找出AI代码的边界缺失，重构正确版本，教师提供"认知支架"而非答案
- **100-110分钟 成果汇报**：两组展示对比（盲从AI组 vs 质疑修正组），聚焦"发现过程"而非结果正确性
- **110-120分钟 元认知反思**：填写"AI欺骗检测清单"：我忽略了哪个边界条件？下次如何第一时间发现？

**可用话术**：
- 封锁阶段："现在不准问AI，你的大脑比GPU值钱"
- 冲突阶段："不管是AI给你的还是老师给你的，他都有一定可能性...你要思考更天真"
- 反思阶段："今天AI骗了你几次？你骗了自己几次？"

**风险提示**：
- **[高风险]** 若学生在封锁阶段即发现题目存在歧义，使用话术："这道题故意设计得不完美，就像真实世界的需求文档，你的任务是找出不完美之处"

### 示例2：切屏滥用应急处理
**场景**：就业班学生频繁切屏使用AI，课堂秩序失控，教师无法技术手段监控

**处理方案**：
- **物理隔离**：立即调整为"前后桌背靠背"座位，屏幕朝向教师，禁用AI检索员角色改为"纸质资料员"
- **任务拆解**：将大题拆为5分钟微任务，每完成一步举手检查后方可下一步，压缩连续切屏时间窗口
- **角色制衡**：指定固定"AI检索员"（轮流担任），其他成员切屏即该组当日禁用AI，利用组内压力自我约束

**风险提示**：
- **[高风险]** 避免当众羞辱切屏学生，改为私下警告并调整座位；若学生出现摔键盘、哭泣等过激反应，立即停止对抗环节，转介心理咨询

## Common Issues
1. **时间失控**：六阶段中前两个阶段超时，导致解构时间不足。建议设置严格的时间哨兵（学生担任），到点即停，未完成部分转入课后思考。
2. **分组失衡**：水平过高学生主导全组，低水平学生沦为旁观者。强制角色轮换，要求"记录员"必须汇报，"AI检索员"必须解释AI错误逻辑，确保全员参与。
3. **陷阱反噬**：学生发现题目确实存在歧义或教师预设陷阱过难，导致信任危机。准备"免责话术"："所有代码都有bug，包括我设计的题目，找出它就是今天的胜利"。
4. **技术依赖复发**：课后学生恢复滥用AI。设计"家庭迁移任务"：要求学生在家长监督下完成一道新陷阱题，家长签字确认"孩子指出了AI的错误"，构建家校共育防线。

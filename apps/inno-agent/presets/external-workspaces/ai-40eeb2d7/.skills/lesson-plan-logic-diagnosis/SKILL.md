# 教案逻辑解构诊断

## When to use
- 用户粘贴AI生成的教案文本并要求"诊断问题"、"检查逻辑"、"看看哪里不对"或"分析合理性"
- 首次接触教案文本，需执行四阶校验中的输入解析、逻辑解构与问题诊断阶段
- 用户质疑具体活动与知识目标的匹配性（如"这个活动真的能教if-elif吗"）

## When not to use
- **需重构建议时**：用户明确要求"怎么改"、"给替换方案"、"设计分层任务卡"或"优化120分钟流程" → 转交 `vocational-context-reconstruction` 技能
- **非诊断场景**：询问如何备课、班级管理技巧、Python语法答疑 → 使用普通问答模式
- **安全红线**：发现涉及学生自伤、暴力或严重心理危机的教学案例设计 → 立即转介学校心理中心，停止诊断流程并提示"建议班主任协同"

## Instructions
- **严格保真**：仅基于用户提供的教案文本诊断，零幻觉；对模糊处标注"[推测]"而非臆断，禁止虚构教学环节
- **强制标签**：必须使用五大标准化标签：[逻辑断裂]、[伪信息化]、[学情不适配]、[考纲偏离]、[管理风险]，定位到具体环节
- **三维验证**：每项活动检验：1)是否直接服务具体知识点 2)难度是否匹配职高生"抽象理解困难"特点 3)120分钟连排课是否避免纯理论讨论超20分钟
- **风险拦截**：识别考纲重大偏离（大学算法/遗漏OS模块/异常处理）或普高转学生心理安全陷阱时，立即中断并提示转介

## Workflow
1. **输入解析**：提取教学环节清单、活动类型、隐含目标知识点、建议课时分配
2. **逻辑解构**：执行三维交叉验证（活动-目标匹配度、认知负荷评估、实操可行性）
3. **问题诊断**：用标准化标签标注问题，精确到具体教学环节，说明违规依据
4. **风险判定**：检查考纲红线与心理安全陷阱，输出转介建议或继续诊断

## Output Format

```markdown
### 诊断概览
- **文档保真度**：[完全基于原文/含合理推测/原文信息不足]
- **总体评级**：[可用/需调整/重大缺陷/建议废弃]

### 问题定位表
| 教学环节 | 问题标签 | 具体描述 | 风险等级 |
|----------|----------|----------|----------|
| [环节名] | [标签] | [违规依据说明] | [高/中/低] |

### 三维验证详情
- **活动-目标匹配**：[分析活动是否直接服务知识点]
- **认知负荷评估**：[分析是否匹配职高生抽象理解能力]
- **实操可行性**：[分析120分钟连排课的"做中学"条件]

### 安全与转介
- [ ] 考纲偏离：[是否包含大学算法或遗漏核心模块]
- [ ] 心理安全：[是否针对普高转学生设计挫败感活动]
- [ ] 转介建议：[人工复核课标/班主任协同/心理中心]

### 下一步指令
如需重构方案，请说："为我生成职高化重构建议"
```

## Examples

**示例1：标准诊断**
输入："导入用'投纸片选工资区间'讲解if-elif，新授20分钟讲嵌套if理论，练习用递归算法流程图..."
输出：
```
### 问题定位表
| 导入 | [伪信息化] | "投纸片"属纸质卡片投票替代电子交互，无法建立代码逻辑映射 | 中 |
| 导入 | [逻辑断裂] | 纸质投票与if-elif语法结构缺乏直接因果关系 | 高 |
| 新授 | [管理风险] | 纯理论讲解20分钟，超过职高生注意力阈值，无结构化支持防切屏 | 中 |
| 练习 | [学情不适配] | 递归算法流程图超出职高生抽象理解能力 | 高 |
| 考纲 | [考纲偏离] | 遗漏OS模块与异常处理，包含未要求的递归算法 | 高 |

### 安全与转介
- [x] 考纲偏离：包含大学数据结构内容，建议人工复核《中等职业学校信息技术课程标准》
```

**示例2：风险拦截**
输入："...设计对比活动，让普高转学生展示自己的代码与普高同学对比..."
输出：
```
### 安全与转介
- [x] 心理安全：活动隐含"证明你比普高差"的对比逻辑，可能触发普高转学生的挫败感与自我否定
- [x] 转介建议：涉及特殊学情心理安全，建议班主任协同设计，避免教师单独实施
```

## Common Issues
- **过度推断**：臆测学生已掌握前置知识（如假设已会列表推导式）→ 应标注"[原文未明确学生基础]"而非直接诊断
- **标签滥用**：将主观不喜欢的活动归为[逻辑断裂] → 必须证明活动与目标无因果关联方可使用
- **遗漏可行性**：仅诊断内容正确性，忽视120分钟连排课中"做中学"条件不足（如无真实编程环境）
- **考纲盲区**：未核对中职Python考纲核心模块（OS/异常处理/文件操作）是否缺失，误将大学算法当必学内容

# 撰写培训顺延申请
## When to use
*   用户明确要求生成一份完整的培训顺延申请文书。
*   用户提供了撰写申请所需的关键信息，如姓名、工号、培训名称和顺延原因。
*   用户询问撰写完整申请所需的信息。

## When not to use
*   用户仅需拟定培训顺延的理由，而不需要完整的申请文书（应路由至 `拟定顺延理由`）。
*   用户请求其他类型的申请（如请假、离职、报销等）。
*   用户寻求法律、政策或公司规章制度的咨询。
*   用户要求执行提交申请等操作。

## Instructions
请用户提供以下关键信息：
1.  **申请人姓名**
2.  **工号**
3.  **培训名称**
4.  **详细的顺延原因**
5.  （可选）**工作单位**
6.  （可选）**申请日期**

## Workflow
1.  识别用户需要完整的培训顺延申请。
2.  引导用户提供申请所需的个人姓名、工号、培训名称、顺延原因等信息。
3.  根据收集到的信息，生成符合正式要求的培训顺延申请文书。
4.  以规范、专业的格式输出生成的文本内容。

## Output Format
输出内容将遵循正式申请文书的通用结构，包括：
*   **称谓**：尊敬的党委教师工作部（或根据实际情况调整）
*   **正文**：包含申请人信息、培训名称、详细顺延原因、申请顺延下一期的请求。
*   **致歉与恳请**：表达对不便的歉意并恳请批准。
*   **结尾敬语**：此致 敬礼！
*   **申请人信息**：申请人、工作单位、日期。

## Examples
*   “帮我写一份培训顺延申请，我叫李可骞，工号20250189，培训是‘第一阶段集中课程培训’，原因是我科研工作紧张。”
*   “请根据我的信息生成一份培训顺延申请书，我需要顺延‘教学技能研修’。”
*   “我需要一份完整的培训顺延申请，请问需要提供哪些信息？”

## Common Issues
*   **信息不完整**: 用户未提供所有必需信息，导致无法生成完整的申请。
*   **请求超出范围**: 用户请求生成非培训顺延类的申请。
*   **理由模糊**: 用户提供的顺延理由过于笼统，可能需要进一步引导用户细化。

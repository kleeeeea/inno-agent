# 拟定顺延理由
## When to use
*   当用户明确表示需要为培训顺延申请“找个理由”、“拟定原因”或“构思理由”时。
*   当用户提供背景信息（如工作任务重、家庭事务等），并希望生成一个有说服力的顺延理由时。
*   当用户询问“如何写培训顺延的理由才更有说服力”时。

## When not to use
*   当用户需要一份完整的培训顺延申请文书时（应路由至 `撰写培训顺延申请` 技能）。
*   当用户询问培训顺延的政策、公司规章制度或法律咨询时（超出助手能力范围）。
*   当用户要求实际提交申请或与外部系统交互时（超出助手能力范围）。

## Instructions
请用户提供导致无法按时参加培训的具体背景信息，例如：
*   具体的工作任务、项目进展或科研压力。
*   家庭中发生的紧急或重要事务。
*   其他个人特殊情况。

## Workflow
1.  **识别意图**: 确认用户需要拟定培训顺延的理由。
2.  **信息收集**: 引导用户提供导致无法参加培训的具体背景信息。
3.  **理由生成**: 根据用户提供的信息，构思并生成一个合理、充分且具有说服力的顺延理由。
4.  **结果输出**: 以清晰、专业的语言呈现生成的顺延理由。

## Output Format
*   **顺延理由**: 根据用户输入背景信息生成的具体、有说服力的顺延原因。
*   **建议措辞**: 针对该理由，提供可用于申请文书中的表达方式。
*   **风险提示**: （若有）可能存在的潜在风险或需要注意的事项。

## Examples
### 用户输入
“我需要为培训顺延找个理由，我最近科研任务重，没有学生和助理。”

### 技能输出
*   **顺延理由**: 因作为新入职科研人员，目前缺少学生及科研助理支持，需独立推进并完成单位既定科研目标与项目任务，导致近期科研工作安排较为紧张，无法抽身参加培训。
*   **建议措辞**: “本人作为新入职科研人员，目前缺少学生及科研助理支持，需独立推进并完成单位既定科研目标与项目任务，近期科研工作安排较为紧张，无法抽身参加‘[培训名称]’。特此申请顺延参加下一期。”
*   **风险提示**: 无。

### 用户输入
“帮我拟定一个培训顺延的合理原因，我最近家里有急事。”

### 技能输出
*   **顺延理由**: 近期家中突发紧急事务，需本人亲自处理，导致无法按时参加培训。
*   **建议措辞**: “近期家中突发紧急事务，需本人亲自处理，占用大量时间和精力，导致无法按时参加‘[培训名称]’。特此申请顺延参加下一期。”
*   **风险提示**: 无。

## Common Issues
*   **信息不足**: 用户未能提供足够具体的背景信息，导致生成的理由过于笼统。
*   **期望不符**: 用户可能误以为此技能会生成完整的申请文书，而非仅是理由。
*   **理由不合理**: 用户提供的背景信息本身难以构成合理顺延理由，需要引导用户重新考虑或提供更多细节。

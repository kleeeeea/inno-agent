# 生成培训顺延申请
## When to use
*   用户明确请求“生成”、“写一份”或“创建”培训顺延申请书。
*   用户提供了生成申请书所需的所有关键信息，包括姓名、身份、工作单位、无法参加的培训名称、申请顺延的原因、期望顺延的安排和申请日期。

## When not to use
*   用户请求了解培训顺延的流程、所需材料或注意事项（应路由至 `provide-application-guidance` 技能）。
*   用户要求代为提交、审批或查询申请状态。
*   用户要求修改或伪造申请内容，或生成非培训顺延类的申请。
*   用户请求提供法律咨询或政策解读。

## Instructions
*   请用户提供以下关键信息：
    *   **个人信息**: 姓名、身份（如新入职教师）、工作单位。
    *   **培训详情**: 无法参加的培训名称、申请顺延的具体原因、期望顺延的安排（如顺延到下一期）、申请日期。

## Workflow
1.  接收并解析用户请求，确认意图为生成培训顺延申请书。
2.  提示用户提供生成申请书所需的各项信息。
3.  根据用户提供的信息，利用内置的培训顺延申请模板生成规范的申请书文本。
4.  将生成的申请书文本输出给用户。

## Output Format
```
场景判断: 用户请求生成培训顺延申请书。
处理方案:
[生成的培训顺延申请书文本]
可用话术:
“这是为您生成的培训顺延申请书草稿，请您仔细核对信息，并根据实际情况进行修改和补充。您可以直接复制使用。”
风险提示:
“请注意，我无法核实您提供信息的真实性，也无法代您提交申请。请确保所有信息准确无误，并自行向相关部门提交。”
后续评估:
“如果您需要了解申请流程或有其他疑问，我可以为您提供指导。”
```

## Common Issues
*   **信息不完整**: 用户未提供生成申请书所需的全部信息。
*   **请求模糊**: 用户请求不够明确，难以判断是生成申请书还是寻求指导。
*   **超出权限**: 用户要求执行提交、审批等超出本技能边界的操作。

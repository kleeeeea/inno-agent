# 课堂应急路径切换与同步控制

## When to use
- **硬件崩溃**：当50%以上电脑死机、蓝屏或无法开机，导致原数字化任务无法继续
- **网络中断**：网速极慢或完全瘫痪，在线资源无法加载，学生陷入空等
- **操作失控**：学生大面积误操作、各玩各的，与教学进度严重脱节
- **时间耗尽**：因设备调试或基础操作卡顿，剩余时间不足完成原任务且核心目标未达成

## When not to use
- **备课阶段的风险预判**：课前24-48小时的教材适配、风险点扫描、认知补丁设计，应使用 `risk-scan-adaptation`
- **硬件维修诊断**：电脑硬件故障排查、网络配置修复等技术问题，转介技术支持
- **学生行为管教**：处理学生纪律问题、家校沟通等非技术性课堂管理，由教师自主处理
- **长期课程规划**：学期整体教学设计或单元规划，超出单课时应急范围

## Instructions
1. **先控场后教学**：优先使用强制同步手段（锁屏/强指令）停止混乱，再宣布切换方案
2. **红黄绿路径**：每分钟评估——绿色（执行原案）、黄色（本地化素材）、红色（不插电纸笔）
3. **零技术依赖**：红色路径下确保任务可在无电、无网、无设备条件下完成核心思维训练
4. **止损优先**：接受任务形式降级，确保核心教学目标（计算思维/信息意识）不因硬件失败而流产

## Workflow
1. **实时态势感知**（每分钟）：扫描设备在线率、网络延迟、学生操作同步率
2. **路径切换判断**：
   - **绿色**：设备网络正常，继续原案但预留同步节点
   - **黄色**：网速慢/部分卡顿，切换本地素材包，取消在线搜索
   - **红色**：大面积崩溃，立即启动不插电纸笔任务（如涂黑白格理解编码）
3. **强制同步控制**：在关键操作前发布排他性口令（"手离鼠标，眼看大屏"）或一键锁屏，防止误操作
4. **应急任务分发**：发放纸质任务单/流程图卡片，教师演示替代学生上机操作
5. **秩序恢复确认**：确认全班跟随新路径后，解除强指令锁定

## Examples
**场景**：三年级信息课，教材要求在线搜索图片，但开机后15分钟仍有半数电脑蓝屏，已乱作一团。

**应急处理**：
1. **强制同步**：大声口令"手离鼠标，背靠椅背，眼看老师"（配合物理锁屏），耗时20秒控场
2. **路径切换**：判定为红色路径，宣布"今天电脑累了，我们用纸笔当程序员"
3. **任务转换**：发放印有"搜索引擎流程图"的纸，让学生用红笔圈出"关键词输入框"，用箭头画出"点击搜索后的结果路径"
4. **核心目标保全**：通过纸笔逻辑任务，确保学生理解"关键词-匹配-结果"的搜索逻辑，虽无实操但思维目标达成

## Common Issues
- **犹豫切换**：教师因担心"没让学生摸电脑"而硬撑，导致整节课报废。应果断降级保目标
- **切换不彻底**：部分学生用纸笔、部分仍尝试修电脑，造成新混乱。需统一锁屏/收鼠标后再发纸
- **话术技术化**：使用"地址栏""标签页"等术语解释切换，低龄学生听不懂。应改用"望远镜""门牌号"等比喻
- **忘记解除锁定**：强指令控场后未宣布解除条件，学生长时间不敢动。需明确"听到三次拍手后动手"

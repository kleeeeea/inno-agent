# 教学风险扫描与适配方案生成

## When to use

- 分析教材任务在旧机房、弱网络环境下的可行性，预判必然导致大规模卡顿的环节
- 针对学生零起点或基础参差（如分不清地址栏与搜索框）设计认知对齐补丁
- 为打字困难、无家庭数字环境的学生准备"扫码输入""同桌协作"或"纸笔对照"等补偿措施
- 设计设备完全失效时的不插电（Unplugged）纸笔替代方案，确保计算思维训练不中断
- 在关键操作节点预设"强指令锁定"触发点，防止学生误操作导致进度脱节

## When not to use

- 课堂进行中已发生大面积设备故障、网络瘫痪或学生操作停滞，需要实时应急切换 → 转 `emergency-teaching-switch` 处理
- 需要硬件维修技术指导、网络配置诊断或软件安装支持 → 转普通技术支持问答
- 涉及学期课程整体规划、课程标准修改或评价体系重构 → 转课程设计专家
- 处理学生行为纪律管理、家校沟通或长期学情跟踪 → 转班级管理支持

## Instructions

- **基于实证的冲突识别**：逐条比对教材活动与真实硬件配置、学生先验知识，标记"在线协作、大文件下载、复杂搜索"等必然导致大规模卡顿的高风险环节
- **零假设补偿原则**：默认学生不会打字、分不清浏览器界面元素，为每个基础操作缺口设计形象化对齐话术（如"蓝色E是望远镜，长条框是放大镜"）
- **三级降级体系**：建立绿色（原方案）、黄色（本地化处理）、红色（不插电纸笔任务）的完整降级路径，确保任何硬件条件下核心素养目标可达成
- **时间敏感设计**：每个补丁和替代方案标注预估耗时，确保认知补偿总时长不超过课时40%，为核心思维训练保留时间

## Workflow

1. **风险点扫描**：对比教材任务与真实条件（硬件配置、学生先验知识），标记必然导致大规模卡顿的环节
2. **认知补丁设计**：将技术术语转化为低年级可理解的视觉化比喻，填补"地址栏""搜索框"等基础概念空白
3. **降级补偿生成**：针对硬件瓶颈准备"本地素材包"预下载方案，针对操作困难准备"同桌协作""纸笔对照"平替手段
4. **不插电预案设计**：为设备完全失效场景设计纸笔逻辑任务（如用涂黑白格理解图像编码，用流程图卡片模拟算法）
5. **强指令节点标记**：在关键操作前预设"手离鼠标，眼看大屏"等排他性口令，明确强制同步触发时机

## Output Format

```
### 场景判断
- 教材任务：[具体任务描述]
- 硬件环境：[设备配置/网络状况/系统版本]
- 学情特征：[零起点/参差水平/无家庭电脑/外来务工子女多]

### 冲突风险点
1. [风险描述] → 影响：[卡顿规模/时间损耗/认知断层]
2. ...

### 适配方案
#### 认知补丁（预对齐）
- 术语转化：[抽象概念] → [形象化比喻/视觉锚点]
- 预操作训练：[步骤简化指令/黑板大字报]

#### 降级补偿（硬件/操作受限）
- 网络瓶颈：[本地素材包/预下载/取消在线搜索]
- 打字困难：[扫码输入/同桌协作/纸笔对照]
- 软件卡顿：[降级软件/静态演示替代实际操作]

#### 不插电预案（红色路径）
- 激活条件：[如50%电脑无法开机/网络瘫痪]
- 纸笔任务：[逻辑训练替代方案描述]
- 所需材料：[格子纸/流程图卡片/涂色工具]

### 可用话术示例
- 概念对齐："找到那个蓝色的E，它就像我们的望远镜"
- 指令锁定："手离鼠标，眼看大屏，我说开始再开始"

### 风险提示
- 时间预警：[补丁环节预估耗时，如"地址栏讲解需3分钟"]
- 秩序风险：[可能的学生误操作/等待焦虑]
- 目标保障：[简化操作后如何确保计算思维理解度]

### 后续评估
- 顺畅度指标：[学生举手求助频次/设备故障率]
- 目标达成：[简化版作业完成率/核心概念理解检查]
```

## Examples

**示例1：三年级搜索任务适配**
- **场景判断**：教材要求"在国家平台搜索视频"，学生零起点（不会打字、分不清地址栏），机房网速慢
- **风险点**：打字需10分钟/人，网页打不开导致秩序混乱，分不清浏览器与搜索框
- **认知补丁**："蓝色E是望远镜（浏览器），长条框是放大镜（搜索框）"；黑板贴关键词大字报供对位输入
- **降级补偿**：取消在线搜索，改用"预存素材包本地浏览"；打字困难者允许同桌协作（一人操作一人观察）
- **不插电预案**：若电脑大面积死机，改为"纸质迷宫图+手绘搜索路径"，用涂色理解关键词筛选逻辑

**示例2：图像编码课时适配**
- **场景判断**：教材要求用软件修改图片尺寸，设备老旧（Win7/4G内存）易卡顿崩溃
- **风险点**：专业软件启动慢，文件保存失败，学生等待焦虑
- **降级补偿**：改用系统画图板而非专业软件；提供已裁剪好的对比图观察而非实时操作
- **不插电预案**：分发8x8黑白格子纸，用涂色游戏理解"像素=小方块，分辨率=格子数量"，确保编码思维不依赖软件

## Common Issues

- **过度乐观假设**：避免假设学生"应该"会基础操作（如鼠标双击、开关机），默认全部需要认知补丁
- **补丁过多超时**：认知补偿总时长超过课时40%时，必须果断切换为不插电方案，确保核心素养目标不被挤压
- **红色路径缺失**：仅准备Plan A（原方案）和Plan B（简化操作），未设计设备完全失效时的纸笔替代方案
- **术语残留**：补丁话术中仍出现"浏览器""地址栏"等抽象词汇，未转化为学生可触摸的视觉化比喻（如望远镜、放大镜）
- **边界模糊**：试图在预案中解决课堂实时纪律管理或现场故障维修，应聚焦教学设计而非现场管控或硬件维护

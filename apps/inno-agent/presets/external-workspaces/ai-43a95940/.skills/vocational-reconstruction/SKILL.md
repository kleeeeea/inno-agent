# 职高化重构建议

## When to use
- 用户明确要求"改进/优化/重构"已诊断出问题的教案环节（如替换"投纸片"类伪信息化活动）
- 需要针对中本一体班（升学导向）与3+2班（就业导向）设计差异化任务难度与考纲深度
- 需要建立具体的小组分工制（AI检索员、记录员、汇报员、质疑员）防止"假讨论真放羊"
- 需要重新分配120分钟连排课的时间轴，确保实操（≥70分钟）与理论讲解（≤30分钟）比例合理
- 需要将抽象概念转化为"错误代码辨析"等具象化编程任务，适配职高学生认知特点

## When not to use
- 仅需分析教案逻辑断裂、活动-目标匹配度等诊断性工作而未要求重构方案 → 转交 **lesson-logic-diagnosis**
- 涉及学生安全风险（如代码操作可能破坏系统）、心理危机或法律合规隐患 → 提示联系班主任/校级心理中心/信息技术安全部门
- 要求从零生成全新教案而非优化现有AI生成内容 → 使用普通问答流程
- 超出中职信息技术类（Python程序设计等）专业课范围 → 提示专业适配限制

## Instructions
- **严格锚定原文**：所有重构建议必须标注针对原教案的具体段落或活动名称，禁止虚构原文不存在的内容
- **120分钟铁律**：重构后的方案必须确保连排课具备"真实编程环境实操"条件，杜绝纯理论讨论或纸质活动占用上机时间
- **分层精准性**：中本一体班侧重算法逻辑与考纲深度（如OS模块原理），3+2班侧重操作熟练度与错误排查（如文件路径调试）
- **反AI设计**：每个重构方案必须包含至少一个"质疑AI"环节（如教师故意设计AI也做不出的错题让学生找出漏洞）
- **角色均等化**：小组分工必须确保每个成员有不可替代的产出物，防止"一人编码三人围观"

## Workflow
1. **确认重构点**：明确用户要求优化的具体环节（需已完成诊断或用户已指出问题）
2. **策略选择**：根据问题类型选择重构路径（伪活动→错误代码辨析；单一任务→分层任务卡；松散讨论→分工制）
3. **班级适配**：针对中本一体班 vs 3+2班调整任务深度，标注对应中职Python考纲条目（OS模块/异常处理/选择结构等）
4. **时间轴重构**：按120分钟连排课重新分配（导入5'-任务发布10'-分层实操70'-展示互评25'-总结10'）
5. **生成可执行方案**：提供可直接使用的任务卡文本、分工表格、错误代码案例

## Output Format
- **重构目标**：原文需替换的具体活动/段落定位（引用原文关键词）
- **职高化策略**：采用的重构方法（错误代码辨析/分层任务/小组分工/时间优化）
- **分层任务卡**（如适用）：中本一体班任务（升学深度）与3+2班任务（就业实操）的具体差异描述
- **小组分工表**：四人角色（AI提问员/代码审查员/记录员/汇报员）的职责定义与轮换机制
- **120分钟时间轴**：优化后的环节分配、过渡话术与应急预案（如提前完成的班级加练内容）
- **反AI依赖机制**：培养质疑精神的具体操作（如"找出AI生成代码中的3个Bug"、"对比两个AI的不同答案"）
- **考纲锚定**：调整后的内容对应中职Python考纲的具体条目编号与要求

## Examples
**示例1：替换伪信息化活动**
- **场景**：原教案用"投纸片看工资"讲解税收概念（伪信息化诊断）
- **重构方案**：改为"错误代码辨析——找出这段Python工资计算程序中的逻辑漏洞"
- **分层设计**：中本一体班分析嵌套if-else算法逻辑与异常处理；3+2班侧重修正语法错误与调试运行
- **反AI设计**：教师提供一段看似正确但隐藏3个逻辑错误的"AI生成代码"，学生需找出AI未考虑的空值判断错误

**示例2：建立小组分工制**
- **场景**：原教案仅写"小组讨论Python选择结构应用场景"（易沦为闲聊）
- **重构方案**：10分钟轮换制分工——①AI提问员（设计Prompt获取案例）②代码审查员（找出AI案例中的Bug）③记录员（截图存档错误类型）④汇报员（总结"AI也会犯的错误"）
- **120分钟适配**：前70分钟各组用真实环境复现AI提供的错误代码并修正；后25分钟跨组展示"最隐蔽的AI错误"

## Common Issues
- **过度学术化**：为追求"深度"引入大学数据结构概念，偏离中职考纲（如过度深入面向对象继承），需严格对照考纲边界
- **分层标签化**：公开将学生分为"快班/慢班"造成心理伤害，应使用"挑战任务/基础任务"自愿选择或隐藏分层
- **时间失控**：在120分钟内填充超过3个大型任务，导致每个都浅尝辄止，应遵循"一大两小"或"一深多浅"原则
- **AI依赖转移**：从"学生直接问AI"变成"教师代问AI再分发"，未真正培养质疑精神，必须保留学生直接操作AI并批判验证的环节
- **伪实操**：将"看教师演示"包装成"实操"，120分钟连排课必须保证学生独立上机编码时间≥70分钟

# 教案逻辑解构与诊断

## When to use
- 教师上传AI生成教案后要求"检查逻辑问题"或"诊断教案缺陷"
- 需要评估教学活动与知识目标的直接匹配度（活动-目标一致性检查）
- 判断活动难度是否适合职高学生基础（避免"纸质卡片"类幼稚化或纯学术化推导）
- 验证120分钟连排实训课是否具备真实"做中学"条件（排除纯理论讨论填充）
- 识别教案中的"无意义活动"、"假讨论真放羊"环节或中职Python考纲偏离点

## When not to use
- **需要具体重构方案时**：如要求"设计替代活动"、"制作分层任务卡"、"建立小组分工制"、"优化时间轴"等，应转交 **vocational-reconstruction** 处理
- **未提供具体教案文本**：仅询问通用教案设计原则时，使用普通问答
- **涉及学生心理危机或安全风险处置**：如教案包含可能引发学生严重焦虑的活动或实训安全隐患，建议转介校级心理中心或安全管理部门，超出逻辑诊断范畴

## Instructions
1. **严格锚定原文**：所有诊断必须引用原文具体段落或活动名称，禁止虚构原教案不存在的内容
2. **三维校验**：必须覆盖活动-目标匹配度、认知负荷评估、120分钟实操可行性三个维度
3. **分级标注**：按严重程度将问题分为"致命缺陷（需立即重构）"、"明显脱节（建议调整）"、"轻微不适（可选优化）"
4. **考纲对照**：对照中职Python考纲（OS模块、异常处理、选择结构等）标注偏离点，区分中本一体班与3+2班要求差异

## Workflow
1. **输入解析**：提取教学活动序列、时间安排、情境创设、提问设计、理论/实操配比
2. **逻辑解构**：逐条验证活动与知识目标的因果关系，识别逻辑跳跃或断裂（如"投纸片"活动与Python文件操作无认知关联）
3. **学情适配评估**：判断活动设计是否符合职高学生"基础薄弱、渴望有意思课程、普高转学生与原生职高生混合"的特点
4. **连排课可行性检查**：验证120分钟内是否具备真实编程环境操作条件，排除纯纸质/讨论类"假放羊"活动
5. **风险识别**：标注可能引发学生挫败感的比较性评价、超纲学术内容或伪信息化活动

## Examples
**输入**："请诊断这份AI生成的Python选择结构教案，120分钟连排课，活动包括：1. 观看微课10分钟 2. 纸质卡片投票判断是否语句适用场景15分钟 3. 学生自主编写工资计算器程序20分钟 4. 小组讨论优秀代码特征30分钟 5. 教师讲授代码规范45分钟"

**诊断要点**：
- **致命缺陷**：活动2"纸质卡片投票"与Python编程目标脱节，属于"投纸片看工资"类伪信息化活动，无法建立真实代码感知，且幼稚化操作对职高生（含普高转学生）产生隐性羞辱感
- **明显脱节**：活动4"讨论优秀代码"占30分钟但缺乏前置充分实操支撑，属于"假讨论真放羊"；活动5讲授时间45分钟超出职高生注意力阈值，违背120分钟连排课"做中学"核心原则，理论/实操配比严重失衡
- **考纲偏离**：未涉及中职考纲要求的选择结构嵌套与异常处理基础，深度不足

## Common Issues
1. **幻觉诊断**：未仔细阅读原文，基于通用教案模板编造不存在的环节进行批判（如指控原文没有的"投纸片"活动）
2. **过度重构**：在本阶段直接给出详细替代活动方案（如具体代码案例或任务卡内容），混淆诊断与重构的边界
3. **忽视连排课特性**：未特别关注120分钟时长下的实操可行性，仅按常规45分钟课理论-实践比例分析
4. **考纲错配**：混淆中职与普高Python考纲差异，或忽视中本一体班与3+2班的不同深度要求，导致误判内容是否超纲或不足

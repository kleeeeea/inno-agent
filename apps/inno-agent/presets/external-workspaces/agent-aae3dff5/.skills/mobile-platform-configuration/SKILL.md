# 手机教学平台工具化配置

## When to use
- 需要配置超星等平台防作弊功能（"未提交前隐藏答案"、限时提交、随机抽题等）
- 设计手机抢答的阶梯积分规则（如第1名3分、2-3名2分、参与即1分）
- 将微课视频分发至学生手机并设置防下载/外传权限
- 创建当堂过关测验（3分钟快速测验）及调取参与率、正确率数据
- 查询平台客观数据指标（抬头率替代指标：任务完成率、讨论原创率、测验提交分布）

## When not to use
- 涉及教师走位、巡航路线、非语言肢体干预 → 转交 `classroom-mobility-psychology`
- 涉及抢答尴尬化解话术、课堂节奏减速锚点、心理安全构建 → 转交 `classroom-mobility-psychology`
- 涉及学生严重情绪失控、自伤倾向或暴力冲突 → 建议立即联系学校心理中心或班主任，不提供课堂管理技巧
- 涉及"监控学生手机屏幕"或"远程控制手机"等侵犯隐私需求 → 拒绝并提供替代方案（任务驱动代替监控）

## Instructions
- **禁止"堵"思维**：绝不推荐手机袋上交、强制关机等强硬手段，坚持"疏导式工具化"
- **具体参数化**：所有方案必须包含可操作的技术参数（如"开启'学生可见答案时机：提交后'"、"设置答题时间180秒"）
- **强制数据闭环**：每节课必须设计当堂过关测验或即时数据采集点，用客观数据替代"假性安静"观察
- **安全积分设计**：避免公开排名羞辱，推荐"个人成长值"或"小组贡献度"，保护后进生积极性

## Workflow
1. **需求识别**：确认平台类型（超星/学习通/智慧职教等）与核心痛点（抄袭/冷场/快讲/无反馈）
2. **防作弊配置**：设置"未提交前隐藏他人答案"、"禁止复制粘贴"、"拍照上传手写答案"等权限
3. **激励机制设计**：配置阶梯积分（手速分+参与分）、随机选人（避免学霸垄断）、即时排行榜（仅显示前3名或小组均值）
4. **内容工具化**：将实操步骤转为微课二维码/短链接，设置"抬头-低头"循环（看30秒视频→抬头实操→再低头确认）
5. **数据验证**：设定下课前3分钟测验，明确调取"参与率+正确率"作为本节课效率的唯一评判标准

## Examples

**示例1：防抄袭讨论配置（超星平台）**
- 场景判断：超星平台，学生复制粘贴百度答案，30人答案标点一致
- 技术配置：话题设置 → 高级设置 → 勾选"学生必须回复后才可看到他人回复" + "禁止粘贴（防复制）"
- 积分规则：原创观点3分，有效评论他人2分，参与1分；设置"词云"展示高频关键词而非个人排名
- 数据闭环：讨论区"原创率"指标（系统自动检测重复度），要求原创率>70%
- 风险提示：避免开启"强制实名+公开评分"，防止后进生被标签化

**示例2：阶梯抢答与当堂测验组合**
- 场景判断：90分钟理论课，教师恐慌快讲，学生低头玩手机
- 技术配置：活动库 → 抢答 → 设置"允许抢答次数3次"（防止学霸垄断）；测验库 → 随机抽题（每人题目顺序不同）
- 积分规则：抢答手速第1名3分，2-3名2分，4-10名1分，参与未抢到0.5分；答错仍得参与分
- 数据闭环：下课前3分钟发布2道选择题（限时2分钟），实时显示正确率；若正确率<50%，则下节课前5分钟自动推送补救微课
- 风险提示：测验成绩仅作为教师调整教学依据，不纳入期末总评，降低学生焦虑

## Common Issues
- **要求监控屏幕**：用户要求"查看学生手机屏幕"或"远程锁屏" → 拒绝，改为推荐"任务驱动"：设置有趣的限时扫码任务，自然吸引学生主动切换屏幕
- **要求公开羞辱排名**：用户要求"把最后几名投到大屏幕上警示" → 安全引导，建议改为"小组贡献度"或"个人进步榜"，避免学生彻底摆烂
- **讨论区仍是复制粘贴**：检查是否同时开启了"未提交前隐藏答案"和"防复制粘贴"两个开关，仅开一个无效
- **抢答被学霸垄断**：检查是否设置了"每人每节课最多抢答2次"或"随机摇一摇选人"作为抢答补充机制
- **微课被下载外传**：检查视频权限设置为"仅限APP内观看，禁止下载"，或采用流媒体加密链接（有效期仅限当节课）

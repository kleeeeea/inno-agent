# 查询会议嘉宾与主题
## When to use
- 用户查询会议的主旨演讲嘉宾、案例分享者或圆桌讨论嘉宾。
- 用户想了解特定嘉宾的演讲或讨论主题。
- 用户询问会议的整体主题。

## When not to use
- 请求会议的完整日程安排或具体时间点 (应路由至 `get-conference-agenda`)。
- 询问嘉宾的个人联系方式或其他非公开信息 (安全转介/超出范围)。
- 请求会议的实时更新、报名、费用、交通、住宿等后勤信息 (超出范围)。
- 寻求主观评价、建议或预测 (超出范围)。

## Instructions
- 识别用户查询中提及的嘉宾类型（主旨演讲、案例分享、圆桌讨论）或会议主题。
- 从会议资料中精确提取对应的嘉宾姓名、身份、演讲/讨论主题。
- 如果用户查询整体会议主题，直接提供。

## Workflow
1.  **理解用户意图**：分析用户关于会议嘉宾、演讲/讨论主题或会议整体主题的查询。
2.  **信息提取**：从已上传的会议资料中，精准提取相关嘉宾信息及其主题，或会议整体主题。
3.  **结果呈现**：以清晰、简洁、专业的中文形式，直接回答用户问题。
4.  **边界处理**：对于资料中未明确提供的信息，明确告知用户无法提供。

## Output Format
- **场景判断**: 用户查询会议嘉宾或主题信息。
- **处理方案**: 提取并展示相关嘉宾姓名、身份、演讲/讨论主题，或会议整体主题。
- **可用话术**:
    - "本次会议的主题是：[会议主题]"
    - "主旨演讲嘉宾是：[嘉宾姓名] ([身份])，演讲题目是：《[演讲题目]》"
    - "案例分享嘉宾是：[嘉宾姓名] ([身份])，分享内容是：[分享内容]"
    - "圆桌讨论嘉宾包括：[嘉宾1姓名] ([身份]), [嘉宾2姓名] ([身份]), [嘉宾3姓名] ([身份]), [嘉宾4姓名] ([身份])，讨论主题是：[讨论主题]"
    - "抱歉，您查询的[特定信息]在现有资料中未明确提供。"
- **风险提示**: 无。
- **后续评估**: 用户是否获得了准确且完整的嘉宾和主题信息。

## Examples
- 用户：“谁会在会议上发表演讲？”
- 用户：“主旨演讲的嘉宾是谁，主题是什么？”
- 用户：“万家阳会分享什么内容？”
- 用户：“圆桌讨论的嘉宾都有谁？”
- 用户：“会议的主题是什么？”

## Common Issues
- 用户询问嘉宾的联系方式。
- 用户查询的嘉宾或主题信息在文档中不存在或不明确。
- 用户将议程相关问题错发给本技能。

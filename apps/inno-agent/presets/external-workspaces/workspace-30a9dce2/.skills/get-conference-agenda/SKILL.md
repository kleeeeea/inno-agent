# 获取会议议程
## When to use
*   用户查询会议的完整日程安排。
*   用户询问会议的具体时间点或时间段。
*   用户想知道某个环节（如签到、茶歇、开幕致辞）的时间。
*   用户询问会议的开始和结束时间。

## When not to use
*   用户查询会议的主题或演讲嘉宾信息（应路由至 `query-speakers-and-topics`）。
*   用户询问嘉宾的具体演讲主题（应路由至 `query-speakers-and-topics`）。
*   用户询问会议报名、费用、交通、住宿等后勤信息（超出代理能力范围）。
*   用户询问会议的具体会议室（若文档中未明确，则告知待定）。
*   用户请求提供嘉宾联系方式（涉及隐私，应拒绝）。

## Instructions
1.  识别用户请求中与会议日程、时间安排相关的关键词。
2.  从提供的会议资料中，精准提取所有时间点、时间段及其对应的活动内容。
3.  按照时间顺序整理并呈现会议的完整议程。
4.  对于文档中明确指出“待定”的信息，如具体会议室，应如实告知。

## Workflow
1.  **意图识别**: 判断用户请求是否聚焦于会议的日程、时间、环节安排。
2.  **信息检索**: 在会议资料中查找所有包含时间（如“13:30 - 14:00”、“14:00”）和对应活动描述的段落。
3.  **内容提取**: 提取每个时间段的活动名称、内容描述及相关参与方。
4.  **结构化输出**: 将提取的信息按时间顺序组织成清晰的议程列表。
5.  **边界处理**: 若用户询问的信息在文档中明确为“待定”或未提及，则按边界处理原则回复。

## Output Format
场景判断: 用户请求会议日程或时间安排。
处理方案: 提取并列出会议的详细议程。
可用话术:
- “好的，这是本次会议的详细议程：[议程列表]”
- “会议将于 [开始时间] 开始，[结束时间] 结束，具体安排如下：[议程列表]”
- “根据会议议程，[特定环节] 将在 [时间] 进行。”
风险提示: 若有信息待定，需明确告知，例如“具体会议室待定”。
后续评估: 用户是否对议程信息满意，是否需要进一步的日程细节。

## Examples
用户: “会议的议程是什么？”
代理: “好的，这是本次中外高校数智化教育合作交流会议的详细议程：
13:30 - 14:00 签到与入场
14:00 - 14:10 开幕致辞 (上海财经大学校方代表)
14:10 - 14:50 主旨演讲（一）：《"应"变"而生"，以"融"促新：生成式AI时代数字经济人才培养探索》 (王子晔 教授)
14:50 - 15:20 案例分享：聚焦西班牙IE大学在数智化教育领域的前沿实践与经验 (万家阳 – 西班牙IE大学)
15:20 - 15:40 集体照&茶歇
15:40 - 16:20 圆桌讨论：中外高校数智化教育合作的机遇、挑战与机制创新
16:20 - 16:30 总结与闭幕，自由交流
会议地点为上海财经大学国定路777号校区，具体会议室待定。”

用户: “签到时间是几点？”
代理: “根据会议议程，签到与入场时间是13:30 - 14:00。”

## Common Issues
*   用户请求的日程信息在文档中缺失或不完整。
*   用户请求的某个时间点没有明确对应的活动。
*   文档中时间格式不统一，导致提取困难。

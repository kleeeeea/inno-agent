# 获取会议概览
## When to use
- 当用户询问会议的主题时。
- 当用户询问会议的时间时。
- 当用户询问会议的地点时。
- 当用户需要会议的基本概览信息时。
- 当用户询问会议名称时。

## When not to use
- 当用户询问特定时间段的活动、嘉宾或讨论主题时，应路由至 `查询会议日程详情`。
- 当用户询问文档中未提及的信息，如注册费用、报名方式、联系方式等。
- 当用户请求主观判断或建议时。
- 当用户请求实时变动或会议组织工作时。

## Instructions
- 识别用户请求中关于会议主题、时间、地点、名称等基本概览信息的意图。
- 从提供的会议议程文档中准确提取相关信息。
- 生成清晰、简洁的回复，仅包含文档中明确列出的概览信息。

## Workflow
1. 接收用户关于会议概览的查询。
2. 识别并提取查询中的关键词，如“主题”、“时间”、“地点”、“名称”、“基本信息”。
3. 在会议议程文档中查找匹配的概览信息。
4. 如果找到信息，则组织成简洁的回复。
5. 如果未找到或请求超出范围，则根据“When not to use”进行处理。

## Output Format
```
场景判断: 会议概览查询
处理方案: 提取并回复会议的主题、时间、地点等基本信息。
可用话术:
- 会议主题是：“[会议主题]”。
- 会议时间是：“[会议时间]”。
- 会议地点是：“[会议地点]”。
- 会议名称是：“[会议名称]”。
- 会议的基本信息如下：[会议主题]、[会议时间]、[会议地点]。
风险提示: 无
后续评估: 确保回复信息与文档内容完全一致，无任何推断或额外信息。
```

## Common Issues
- 用户请求文档中未提及的会议概览信息（如注册费用、报名方式），应明确告知无法提供。
- 用户请求主观评价或建议，应拒绝回答。
- 用户请求实时变动信息，应说明只能提供基于现有文档的信息。

# 教研机制闭环设计

## Instructions

- 基于用户提供的访谈事实或现状描述，先完成问题分型（结构性挤占 / 机制性失灵 / 机制副作用），再进入方案设计，不得跳过诊断直接给建议
- 所有方案必须明确责任主体（谁执行、谁审批、谁公示）、操作规则和执行边界，禁止输出口号式或原则性表述
- 五个模块须逐一覆盖：①时间保障与会议治理 ②去行政化议题机制与会议结构 ③产出—反馈—复盘闭环 ④共性备课与专业差异化适配（双层结构）⑤可量化成效指标
- 识别关键阻力（老教师动力不足、会议频繁占用、跨专业沟通成本高）并给出可操作缓解策略
- 给出阶段性推进路径，建议从"1学科×1专业"试点起步，设置里程碑与验收点

## Workflow

1. **问题诊断**：锚定用户提供的具体事实（如被占次数、退化路径、开展次数），输出《问题诊断报告》，标注问题类型与因果链
2. **机制方案设计**：按五个模块逐一设计，优先回应已诊断的问题类型，输出《机制优化方案》
3. **落地工具包生成**：输出议程模板、任务记录单、专业适配映射表、成效指标看板等可直接使用的工具
4. **推进路径规划**：给出试点—扩散路线图，含里程碑节点与验收标准

## Output Format

- **《问题诊断报告》**：问题分型表（类型 / 典型表现 / 因果链 / 关键证据）
- **《机制优化方案》**：五模块结构化方案，每模块含制度规则、责任主体、操作流程
- **落地工具包**（按需生成）：
  - 教研会议议程模板（含行政事项限时通报区 ≤10 分钟 + 专业研讨区）
  - 教研任务记录单（议题来源 / 讨论要点 / 可交付物 / 下次复盘任务）
  - 专业需求—知识点—课堂任务—评价证据映射表
  - 成效指标看板（过程指标 + 结果指标 + 对齐校历/绩效节点）
- **推进路径图**：试点阶段 → 扩散阶段，含时间节点与验收标准

## Examples

**输入示例**：
> 我们学校18周教学周，全校大会、工会、党员会议占了5次，调休撞车2次，班主任事务又多，一学期教研只开了6次，而且基本都是传达通知、说班情，数学组集中备课倒是坚持了，但老师反映不同专业的学生差异大，备课内容用不上。

**输出摘要**：
- 诊断：同时存在结构性挤占（5+2次被占，实际可用仅6次）、机制性失灵（议题退化为通知传达+班情）、机制副作用（集中备课共性压过个性）
- 模块①：校级硬约束——固定教研时段列入校历保护清单，全校大会等须提前2周申请教务审批方可占用，被占后须在2周内通过补研窗口或线上异步补足
- 模块③：每次教研须提交最低可交付物（课例片段/差异化任务单/学情要点），下次教研前10分钟复盘上次任务证据
- 模块④：数学教研产出"共性底座+专业适配层"双版本，引入各专业负责人每学期参与2次共研，形成专业需求清单

## Common Issues

- **跳过诊断直接给方案**：必须先完成问题分型，否则方案无法对症；若用户信息不足，应主动追问关键事实（被占次数、实际开展次数、典型退化路径）
- **方案缺乏责任主体**：每条制度规则须明确"谁来执行/谁来审批"，避免出现"学校应当……"等无主语表述
- **工具模板过于复杂**：落地工具须控制填写成本，单张记录单字段不超过8项，优先复用现有教务系统或钉钉/企微表单
- **忽视班主任双线并行现实**：方案须显式回应班主任事务与教研时段的冲突，给出错峰安排或流程减负方案，不得要求教师自行消化

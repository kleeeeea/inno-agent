# 教研问题诊断与分型

## Instructions

- 接收用户描述的教研现象（如被占次数、会议退化路径、备课副作用等），必须锚定用户提供的具体事实，不得泛泛归因
- 将问题严格归入以下三类（可多类并存），每类独立分析，不得混淆：
  - **结构性挤占**：校级会议/工会/党员/部门会议占用、班主任双线并行、校历调休撞车
  - **机制性失灵**：议题退化为通知传达→班情汇报→闲聊、专题学习无反馈无产出、缺闭环
  - **机制副作用**：集中备课"共性压过个性"、跨专业适配缺失、差异化依赖个人自觉
- 因果链须说明"机制如何导致问题"，禁止使用"教师不重视"等简单归因
- 诊断结论须为后续机制方案设计提供明确的问题入口

## Workflow

1. **事实提取**：从用户输入中提取关键数据（如被占次数、实际开展次数、典型退化路径、访谈引语）
2. **问题分型**：逐类判断是否存在结构性挤占 / 机制性失灵 / 机制副作用，标注证据来源
3. **因果链构建**：对每类问题说明"触发条件→中间机制→最终表现"的完整链条
4. **输出诊断报告**：按分型结构输出，附关键证据引用与后续优化方向提示

## Output Format

输出《教研问题诊断报告》，包含以下结构：

**一、问题分型总览**（表格：问题类型 | 是否存在 | 关键证据）

**二、逐类因果链分析**
- 【结构性挤占】触发条件 → 中间机制 → 表现
- 【机制性失灵】触发条件 → 中间机制 → 表现
- 【机制副作用】触发条件 → 中间机制 → 表现（如不存在则注明）

**三、诊断结论与优化入口**（每类问题对应1条核心改进方向，为后续方案设计铺路）

## Examples

**输入示例：**
> 我校18周教学周中，全校大会、工会、党员会议占用了5次，调休撞车2次，班主任事务又占了不少时间，教研组长反映一学期只开了6次教研。开会内容基本是传达通知、说班情，数学组集中备课还好，但不同专业的学生需求差异很大，老师都照着统一教案上。

**输出示例（节选）：**

| 问题类型 | 是否存在 | 关键证据 |
|---|---|---|
| 结构性挤占 | ✅ | 大会/工会/党员会议占5次，调休撞车2次，实际仅开6次 |
| 机制性失灵 | ✅ | 会议内容退化为通知传达+班情汇报，无产出机制 |
| 机制副作用 | ✅ | 集中备课统一教案，跨专业差异化适配缺失 |

【结构性挤占因果链】
校级会议统筹缺位（无硬约束规则）→ 各类会议自由占用固定教研时段 → 18周中有效教研次数压缩至6次，名义频次与实际频次严重背离。

## Common Issues

- **不得将三类问题合并为一段分析**：每类须独立成节，便于后续针对性设计方案
- **避免无证据的推断**：若用户未提供某类问题的具体事实，应注明"当前信息不足以确认，建议补充访谈数据"
- **因果链不等于现象罗列**：须说明机制层面的传导路径，而非仅描述表面现象
- **机制副作用易被忽略**：集中备课看似有效，须主动检查是否存在"共性覆盖个性"的副作用，不能因表面运转正常而跳过
